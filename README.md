# 📚 EZ-BookReturn | 校園二手書收購平台

這是一個專為校園設計的一站式二手書收購預約網站，幫助學生快速估算二手書價值，並提供自動化管理後台供管理者進行數據分析與報表匯出。

## ✨ 核心功能 (Features)

### 📢 學生端（使用者介面）
* **💰 即時估價計算機**：透過 JavaScript 邏輯，根據「全新、良好、差強人意」三種書況即時試算總收購金額。
* **📅 自動化預約系統**：內建表單防呆驗證（格式檢查、必填校驗），資料透過 Fetch API 即時同步至 Google 表單。
* **✉️ 聯絡與提問**：獨立的諮詢管道，支援學生針對預約時段進行修改或取消。

### 📊 管理端（後台系統）
* **🔐 資安驗證登入**：設有管理員密碼驗證機制，防止未授權存取。
* **📈 數據視覺化**：整合 **Chart.js**，以直覺的圖表顯示各時段的預約人數分佈。
* **📥 Excel 簽到表匯出**：整合 **SheetJS (XLSX)**，支援一鍵產生帶有簽名欄位的 `.xlsx` 報表，解決 PDF 中文編碼問題並提升行政效率。

## 🛠️ 技術架構 (Tech Stack)

* **前端 (Frontend)**：
    * **核心技術**：HTML5, CSS3, Vanilla JavaScript (ES6+)。
    * **UI/UX**：響應式設計 (RWD)、Google Fonts 整合。
    * **第三方庫**：Chart.js (數據圖表)、SheetJS (Excel 處理)。
* **後端 (Backend/API)**：
    * **資料庫**：Google Sheets / Google Forms。
    * **API 中繼站**：**Google Apps Script (GAS)**。實作 Serverless API 進行資安過濾，確保敏感個資（如學號、電話）不暴露於前端代碼。

## 🚀 專案亮點 (Highlights)

1. **資安過濾機制**：不直接開放試算表讀取權限，而是透過 GAS 撰寫 API 中繼邏輯，僅回傳去個資化（De-identified）後的統計資料。
2. **零成本部署方案**：全專案採用 Serverless 架構，無需租用伺服器或資料庫，利用 Google 生態系達成商用等級的營運邏輯。
3. **自動化報表流**：解決校園面交現場的行政痛點，實現從「線上預約」到「線下簽到表下載」的完整自動化閉環。
